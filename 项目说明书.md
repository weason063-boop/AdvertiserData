# 账单计费系统 — 项目说明书

> **项目名称**: Antigravity Manager 计费系统
> **最后更新**: 2026-03-26

---

## 一、项目概述

本项目是一套**广告代理服务费自动计算与管理系统**，主要用于：

1. **自动计算服务费** — 根据客户合同条款，结合消耗明细，自动计算代投/流水服务费
2. **合同条款管理** — 支持从飞书多维表格同步合同数据，或手动上传 Excel / 数据库管理
3. **数据看板** — 可视化展示月度/季度消耗、服务费统计、客户排名、趋势分析、智能洞察
4. **汇率维护** — 手工录入恒生银行外汇牌价，用于多币种消耗的 USD 换汇
5. **飞书→金蝶集成** — 将飞书审批流（付款/报销）转换为金蝶凭证 Excel
6. **操作审计** — 全链路操作日志（计算、上传、汇率录入等），支持查询与导出

## 四、数据库模型

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | Integer (PK) | 自增主键 |
| `name` | String (UNIQUE) | 客户名称 |
| `business_type` | String | 业务类型 |
| `department` | String | 所属部门 |
| `entity` | String | 主体 |
| `fee_clause` | String | 服务费条款 |
| `created_at` | DateTime | 创建时间 |
| `updated_at` | DateTime | 更新时间 |

### 4.2 `client_contract_lines` — 合同条款行

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | Integer (PK) | 自增主键 |
| `source_type` | String | 来源类型 (feishu/excel) |
| `source_token` | String | 来源标识 |
| `source_row_index` | Integer | 来源行号 |

| `department` | String | 部门 |
| `entity` | String | 主体 |
| `fee_clause` | String | 条款内容 |

| 字段 | 类型 | 说明 |

## 七、核心业务逻辑

### 7.1 计费引擎 (`billing/`)

已从单文件 `calculate_service_fee.py` 重构为模块化包：

| 模块 | 功能 |
|------|------|
| `fee_engine.py` | 主计算流程：加载消耗→换汇→匹配条款→计算费用→输出 Excel |
| `clause_parser.py` | 条款解析：百分比/固定费/阶梯费率/时间感知条款 |
| `contract_loader.py` | 合同加载：飞书多维表格 / SQLite 数据库 / Excel 文件 |
| `client_overrides.py` | 客户特殊规则：前置/后置覆盖 (从 `client_overrides.json` 加载) |

**计算规则**：
1. 判断服务类型（代投 / 流水），条款无对应类型则费用为 0
2. 只有 **Google 流水** 才计算服务费，TT/FB 等其他媒介流水 = 0
3. 支持百分比费率和固定费用两种模式
4. 支持阶梯费率（如：消耗 ≤3W 收 10%，>3W 收 8%）
5. 支持时间感知条款（如：2025年9月起 FB 5%）
6. 支持合计消耗触发（如：代投合计消耗 ≤X 则 Y%）

**支持的媒介**：Google (GG)、Facebook (FB)、TikTok (TT/TTD)、Apple (ASA)、Kwai、Snapchat (SC)、X (Twitter)、Reddit、LinkedIn、Line、Pinterest、Bing

**合同条款来源**（按优先级）：
1. 飞书多维表格（远程同步）
2. SQLite 数据库（本地存储）
3. Excel 文件（手动上传）

### 7.2 多币种换汇

系统支持 USD / RMB / JPY 三种币种的消耗数据，非 USD 消耗在计算前统一折算为 USD：

| 币种 | 换汇公式 | 数据来源 |
|------|----------|----------|
| USD | 1.0 (基准) | — |
| RMB → USD | `CNY 电汇买入 / USD 电汇卖出` | 恒生银行手工录入快照 |
| JPY → USD | `JPY 电汇卖出 / USD 电汇买入` | 恒生银行手工录入快照 |

汇率数据通过 **手工录入日快照** 维护，存储在 `uploads/hangseng_daily_fx_state.json`。每次计算时自动读取当日快照，若当日无快照且存在 RMB/JPY 消耗则阻断计算。

### 7.3 飞书集成

- **合同同步** (`fetch_feishu_contracts.py`): 从飞书多维表格拉取客户合同条款
- **审批转凭证** (`feishu_to_kingdee.py`): 将飞书付款/报销审批数据转为金蝶凭证 Excel

### 7.4 差异对比 (`compare_by_client.py`)

按客户维度对比自动计算结果与手工计算结果，生成差异报告，用于验证计算准确性。

## 九、登录与安全

### 默认超级管理员

| 项目 | 值 |
|------|------|
| **用户名** | 从 `.env` 的 `ADMIN_USERNAME` 读取 |
| **密码** | 从 `.env` 的 `ADMIN_PASSWORD` 读取 |
| **角色** | `super_admin` |

- Token 有效期：**24 小时**
- 认证方式：OAuth2 Bearer Token (JWT)
- 密码加密：bcrypt 哈希

> 首次启动时如果不存在 `super_admin` 角色的用户，系统会从环境变量自动创建。

## 十一、部署与启动

### 11.1 Windows Service 部署 (推荐)

使用 NSSM 注册为 Windows 服务，开机自启：

```bat
:: 以管理员身份运行
install-services.bat
```

安装后的服务：

| 服务名 | 端口 | 说明 |
|--------|------|------|
| `billing-backend` | 8000 | FastAPI 后端 (uvicorn) |
| `billing-frontend` | 5173 | 前端静态文件服务 |

管理服务：
```powershell
# 查看状态
Get-Service billing-backend, billing-frontend

# 重启 (需管理员权限)
Restart-Service billing-backend -Force
```

### 11.2 本地开发启动

**方式一：一键启动 (`start.bat`)**

```bat
双击 start.bat
```

**方式二：手动启动**

```bash
# 终端 1：启动后端
cd 账单
python -m uvicorn api.main:app --host 0.0.0.0 --port 8000

# 终端 2：启动前端
cd 账单/web
npm run dev -- --host 0.0.0.0
```

### 11.3 Docker 部署

```bash
cd 账单
docker-compose up -d
```

Docker Compose 包含 2 个服务：

| 服务 | 容器名 | 端口 |
|------|--------|------|
| `backend` | billing-backend | 8000 |
| `frontend` | billing-frontend | 80 |

### 11.4 访问地址

| 环境 | 前端地址 | 后端地址 |
|------|----------|----------|
| 本地开发 | `http://localhost:5173` | `http://localhost:8000` |
| Docker 部署 | `http://localhost` | `http://localhost:8000` |
| 局域网访问 | `http://<本机IP>:5173` | `http://<本机IP>:8000` |
